# Phase 15 x86 Boot Info Relocation to 0x7000 Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Align x86 Stage2 behavior with the design requirement that `boot_info` is built at physical address `0x7000`.

**Architecture:** Replace static in-image boot_info pointer usage with explicit low-memory boot_info construction at `0x7000`. Zero the region, initialize magic fields, pass that pointer to `kernel_main`, and surface address in smoke markers for deterministic verification.

**Tech Stack:** GNU binutils cross toolchains, Bash, GNU Make, QEMU, pytest
